Propósito del Curso

Este curso lleva el desarrollo web al siguiente nivel. Ya conocés las bases — ahora las vamos a dominar.

Durante 7 semanas trabajaremos con las herramientas y técnicas que usan los desarrolladores web profesionales hoy: diseño responsive, CSS avanzado, JavaScript para interfaces interactivas y Bootstrap 5.

Cada clase combina teoría con práctica inmediata. Al final del módulo tendrás un proyecto completo publicado en internet.

🏁 Objetivo General

Al finalizar el curso serás capaz de diseñar, desarrollar y publicar sitios web responsive y funcionales, aplicando HTML5 semántico, CSS avanzado, JavaScript vanilla y Bootstrap 5.

¿Qué cubriremos?

  • HTML HTML5 semántico y accesibilidad
  • CSS Variables, BEM, Flexbox, Grid
  • CSS Responsive · Media Queries · clamp()
  • FW Bootstrap 5 · Componentes y Grid
  • JS DOM · Eventos · Fetch API

Frontend vs Backend

🖥️ Frontend

Lo que el usuario VE e INTERACTÚA

  • HTML — estructura del contenido
  • CSS — estilos y presentación visual
  • JavaScript — comportamiento e interactividad
  • Corre en el navegador del usuario
  • Se preocupa por: diseño, UX, accesibilidad, responsive

⚙️ Backend

Lo que ocurre en el SERVIDOR

  • Lenguajes: Python, PHP, Node.js, Java...
  • Bases de datos: MySQL, PostgreSQL, MongoDB
  • APIs que entregan los datos al frontend
  • Corre en el servidor
  • Se preocupa por: seguridad, rendimiento, lógica de negocio
🎯 Nuestro enfoque: Este curso cubre el 100% del Frontend. Vamos a dominar HTML, CSS y JavaScript — las tres tecnologías que construyen todo lo que el usuario ve en la web.

VS Code — Nuestro Editor

Visual Studio Code es el editor de código más popular del mundo para desarrollo web. Es gratuito, liviano y extremadamente poderoso gracias a sus extensiones.

Lo desarrolla Microsoft y tiene soporte oficial para HTML, CSS y JavaScript — exactamente lo que necesitamos.

⬇️ Descargar VS Code

🧩 Extensiones Esenciales

🌐

Live Server

Abre tu HTML en el navegador y lo recarga automáticamente cada vez que guardás. Esencial para ver cambios en tiempo real.

🎨

Prettier — Code Formatter

Formatea automáticamente tu código para mantenerlo ordenado y legible. Se activa al guardar.

💡

IntelliSense for CSS

Autocompletado inteligente para clases CSS dentro de tus archivos HTML. Evita errores de tipeo.

🔵

HTML CSS Support

Sugiere clases CSS definidas en tus archivos de estilo mientras escribís HTML. Ahorra tiempo.

🌈

Color Highlight

Muestra una vista previa del color directamente en el código. Útil al trabajar con paletas CSS.

⚡

Auto Rename Tag

Cuando renombrás una etiqueta HTML de apertura, la de cierre se actualiza automáticamente.

📦

Bootstrap 5 Quick Snippets

Atajos de teclado para insertar componentes Bootstrap sin tener que memorizar todas las clases.

🐙

GitLens

Integración avanzada con Git directo en VS Code. Ver historial, quién modificó qué línea y cuándo.

💡 Atajo clave: Para instalar extensiones, presioná Ctrl + Shift + X y buscá por nombre.
04

Git — Control de Versiones

🎬 Antes de empezar — mirá este video

¿Qué es Git?

Git es un sistema de control de versiones distribuido. Registra todos los cambios que hacés en tu código a lo largo del tiempo, permitiéndote volver a cualquier versión anterior.

Fue creado por Linus Torvalds en 2005 — el mismo creador de Linux. Hoy es el estándar absoluto de la industria.

🕹️ Analogía: Git es como el botón de "guardar partida" en un videojuego. Podés guardar en cualquier momento y si algo sale mal, volvés al punto de guardado anterior sin perder nada.

Git guarda tu historial localmente en tu máquina. GitHub / GitLab lo guardan en la nube.

📚 Conceptos Clave

Repositorio (repo)

La carpeta de tu proyecto con todo el historial de cambios. Puede ser local (en tu PC) o remoto (en GitHub).

Commit

Un "punto de guardado" con un mensaje descriptivo. Registra qué cambió, quién y cuándo.

Branch (rama)

Una línea de trabajo independiente. Permite desarrollar funcionalidades sin afectar el código principal.

Merge

Unir los cambios de una rama con otra. Así se integran nuevas funciones al proyecto principal.

Push

Enviar tus commits del repositorio local al repositorio remoto (GitHub).

Pull

Descargar los cambios del repositorio remoto a tu máquina local.

Clone

Copiar un repositorio remoto completo (con todo su historial) a tu máquina local.

Stage (Index)

Área intermedia donde seleccionás qué archivos incluir en el próximo commit.

🐙 Paso previo: Crear tu cuenta en GitHub

Antes de instalar Git, necesitás tener una cuenta en GitHub — la plataforma más grande del mundo para alojar repositorios y colaborar en proyectos de código.

  • 1
    Ir a github.com Abrí tu navegador y visitá https://github.com/
  • 2
    Hacer clic en "Sign up" Botón ubicado en la esquina superior derecha de la página.
  • 3
    Ingresar tu correo electrónico Usá un correo al que tengas acceso. Lo necesitarás para verificar la cuenta.
  • 4
    Crear una contraseña segura Mínimo 8 caracteres, combinando letras, números y símbolos.
  • 5
    Elegir tu nombre de usuario Este nombre será visible públicamente en todos tus repositorios. Usá algo profesional, por ejemplo: jmontes-dev. No uses espacios ni caracteres especiales.
  • 6
    Verificar que no sos un robot 🤖 Completar el captcha de verificación que aparece en pantalla.
  • 7
    Confirmar tu correo electrónico GitHub te enviará un código de 6 dígitos al correo. Ingresalo para activar tu cuenta.
  • 8
    ¡Listo! Ya tenés tu cuenta Podés personalizar tu perfil en Settings → Profile. Agregá tu nombre real, foto y una breve descripción.
💡 Recomendaciones para tu usuario de GitHub: Elegí un nombre corto, profesional y fácil de recordar. Este usuario aparecerá en la URL de todos tus proyectos: github.com/tu-usuario. Muchos reclutadores revisan perfiles de GitHub, ¡hacelo presentable!

⬇️ Instalación de Git

  • 1
    Descargar Git Ir a git-scm.com/downloads y descargar el instalador para Windows.
  • 2
    Ejecutar el instalador Dejar todas las opciones por defecto. En "Default editor" podés seleccionar VS Code.
  • 3
    Verificar la instalación Abrir Git Bash o la terminal y ejecutar el siguiente comando:
Terminal / Git Bash

⚙️ Configuración Inicial (una sola vez)

Terminal — Registrar tu identidad
# Registrar tu nombre (aparecerá en cada commit)
git config --global user.name "Tu Nombre Completo"

# Registrar tu correo
git config --global user.email "[email protected]"

# Verificar la configuración
git config --list

🛠️ Comandos Principales

Flujo completo de trabajo
# 1. Iniciar un repositorio en la carpeta actual
git init

# 2. Ver el estado de tus archivos
git status

# 3. Agregar archivos al stage (preparar para commit)
git add index.html            # un archivo específico
git add .                   # todos los archivos

# 4. Guardar un commit con mensaje
git commit -m "Primer commit: estructura inicial del proyecto"

# 5. Conectar con repositorio remoto en GitHub
git remote add origin https://github.com/usuario/mi-repo.git

# 6. Subir los commits a GitHub
git push -u origin master

🌿 Trabajando con Ramas

# Ver todas las ramas
git branch

# Crear una nueva rama
git branch nombre-de-la-rama

# Cambiar a una rama
git checkout nombre-de-la-rama

# Crear rama y cambiar a ella en un solo comando
git checkout -b nueva-funcionalidad

# Unir (merge) una rama con la principal
git checkout master
git merge nueva-funcionalidad
💡 Buena práctica para mensajes de commit: Usa verbos en presente y sé descriptivo. Ejemplo: "Agrega sección de contacto con formulario" — no solo "cambios".
⚠️ Importante: Nunca hagas commit de contraseñas, tokens o archivos con información sensible. Usá un archivo .gitignore para excluirlos.

Tarea

📋 Página HTML con Estructura Semántica

Crear una página HTML completa usando las etiquetas semánticas de HTML5. El tema es libre — puede ser sobre vos, un hobby, un deporte o cualquier tema de interés.

  • Usar estructura semántica: <header>, <nav>, <main>, <section>, <article>, <footer>
  • Incluir al menos: un encabezado, navegación, 2 secciones de contenido y pie de página
  • Agregar al menos una imagen con atributo alt correctamente descriptivo
  • Incluir una lista (ordenada o desordenada) y una tabla
  • Aplicar CSS básico en un archivo style.css externo
  • Subir el proyecto al repositorio de GitHub creado en clase
  • El mensaje del commit debe ser descriptivo: "Clase 01 - tarea: página HTML semántica sobre [tema]"
